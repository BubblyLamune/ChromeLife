What happened to the notification options?

In Chrome 28 for Windows and ChromeOS, the Chrome team decided to remove the webkitNotifications.createHTMLNotification API, which was removed from the official spec for web pages months earlier, but had previously been stated would remain available for extensions to use. The feature is slated to be removed from Mac and Linux soon as well.

HTML notifications were what allowed you to customize the appearence of notifications to your liking, such as displaying someone's location in the corner, adding colors, or graying out images. There is still a plain text notification API available, which is what's currently being used. It does not allow for any customizations.

Looking to the future

Also in Chrome 28 for Windows and ChromeOS, a new "rich" notification API was introduced. It allows for certain specific deviations from plain text, but still does not offer anywhere near the amount of customizability as HTML notifications did. Support for these notifications within ChromeLife is planned for the near future. Until then, plain text will be the only option available to users of Chrome 28 on Windows or ChromeOS. HTML notifications will still be the preferred method in any browser that still supports them.

Name formats

Name formats are a simple way to specify you would like a user's name to appear. The following options are supported:

  • %U - The user's legacy name, with "Resident" trimmed off
  • %UU - The user's full legacy name, including "Resident"
  • %u - The user's lowercase legacy name, with "resident" trimmed off
  • %uu - The user's full lowercase legacy name, including "resident"
  • %F - The user's first name
  • %f - The user's first name, lowercase
  • %L - The user's last name, with "Resident" trimmed off
  • %LL - The user's last name, including "Resident"
  • %l - The user's lowercase last name, with "resident" trimmed off
  • %ll - The user's lowercase last name, including "resident"
  • %D - The user's display name
